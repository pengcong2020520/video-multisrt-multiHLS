# AI 短剧多语种翻译与音轨外切平台 PRD

## 1. 产品定位

本产品是面向内部运营人员的短剧本地化生产工具。它不是公开视频托管平台，也不是完整剪辑软件。MVP 聚焦单条 1-3 分钟短剧视频的多语种字幕和音轨生成。

产品形态采用 Web 前端 + Agent Runtime + Skill 的组合。Web 负责上传、配置、校对、预览和下载；Agent Runtime 负责任务编排、上下文传递、失败重试和人工暂停点；Skill 负责可复用的具体能力，例如音频拆分、声源分离、ASR、翻译、TTS、混音和打包。

## 2. 核心用户旅程

1. 运营创建项目并上传无字幕原视频。
2. 运营选择源语言、目标语言、翻译风格和配音配置。
3. Agent Runtime 根据处理模板创建执行计划。
4. Media Skill 自动提取音频，分离人声和背景音。
5. ASR Skill 对人声做识别，生成原文台词、时间戳和可选说话人。
6. Localization Skill 调用 DeepSeek 或其他翻译模型，生成短剧本地化译文。
7. 运营在校对页检查原文、译文、角色和时间轴。
8. Subtitle/TTS Skill 生成目标语言字幕和目标语言 TTS 人声音轨。
9. Mixing Skill 将目标语言人声与原背景音混合，生成目标语言音轨。
10. 运营在网页播放器中切换字幕和音轨预览。
11. 运营下载字幕、音轨、预览 MP4 或完整结果包。

## 3. 页面与功能

### 3.1 项目列表页

#### 功能

- 展示视频项目列表。
- 支持按状态、源语言、目标语言、创建时间筛选。
- 支持查看处理进度、失败原因和最近更新时间。
- 支持进入项目详情。

#### 字段

| 字段 | 说明 |
| --- | --- |
| 项目名称 | 默认使用上传文件名，可编辑。 |
| 视频时长 | MVP 建议限制 1-3 分钟，超出时提示风险。 |
| 源语言 | 中文、英文或自动识别。 |
| 目标语言 | 英文、西语、葡语、中文。 |
| 状态 | 上传中、处理中、待校对、生成中、完成、失败。 |
| 创建人 | 内部运营账号。 |
| 更新时间 | 最近一次任务状态变化时间。 |

### 3.2 上传与配置页

#### 必填配置

- 上传视频文件。
- 源语言：自动识别、中文、英文。
- 目标语言：英文、西语、葡语、中文。
- 翻译风格：短剧口语化、强情绪、平台自然口吻。
- 配音模式：默认音色、按角色分配音色、暂不生成配音。

#### 可选配置

- 是否启用说话人识别。
- 是否启用人声分离。
- 是否生成 MP4 预览文件。
- TTS 服务商：默认、MiniMax、豆包、开源本地模型。
- 术语表或角色设定。
- 处理模板：仅字幕、字幕加配音、完整混音打包。

#### 校验规则

- MVP 优先支持 MP4/MOV。
- 单视频建议 <= 3 分钟。
- 文件大小上限由部署环境配置，建议 MVP 先设为 500MB。
- 无音轨视频不可进入 ASR/TTS 流程。
- 如果用户关闭人声分离，只生成字幕和独立 TTS 人声音轨，不承诺替换原人声。

### 3.3 处理进度页

#### 展示内容

- 当前阶段。
- 当前 Agent Run 和 Skill Run。
- 阶段进度。
- 耗时。
- 错误信息。
- 可重试操作。

#### 阶段定义

| 阶段 | 说明 |
| --- | --- |
| uploaded | 文件已上传。 |
| planning | Agent Runtime 正在生成执行计划。 |
| probing | 解析视频元信息。 |
| extracting_audio | 提取原始音频。 |
| separating_sources | 分离人声和背景音。 |
| transcribing | 识别原语言台词。 |
| segmenting | 分句、合并短句、修正时间轴。 |
| translating | 生成目标语言译文。 |
| proofreading | 等待人工校对。 |
| synthesizing | 生成目标语言人声。 |
| mixing | 混合目标语言人声和背景音。 |
| packaging | 生成字幕、音轨、预览 MP4 和下载包。 |
| completed | 处理完成。 |
| failed | 处理失败。 |

### 3.4 校对编辑页

#### 核心能力

- 展示原文、译文、开始时间、结束时间、说话人。
- 支持按目标语言切换。
- 支持修改译文。
- 支持修改说话人和目标音色。
- 支持合并/拆分句段。
- 支持单句试听 TTS。
- 支持单句或单语言重跑翻译/TTS。

#### 表格字段

| 字段 | 说明 |
| --- | --- |
| segment_id | 句段唯一 ID。 |
| start_ms | 开始时间。 |
| end_ms | 结束时间。 |
| speaker_id | 说话人 ID，可为空。 |
| source_text | ASR 原文。 |
| translated_text | 目标语言译文。 |
| voice_id | TTS 音色。 |
| tts_status | 未生成、生成中、完成、失败。 |
| quality_flags | 过长、疑似错译、时间过短等提示。 |

#### 交互原则

- 修改译文后，对应字幕立即更新草稿。
- 修改译文不会自动重跑整条视频，需要用户点击生成或重跑。
- 合并/拆分句段会影响字幕和 TTS，对用户显示明确提示。
- 已完成结果支持生成新版本，不覆盖历史版本。

### 3.5 网页预览页

#### 必须支持

- 播放原视频。
- 切换字幕：无字幕、原文字幕、英文、西语、葡语、中文。
- 切换音轨：原音轨、目标语言混音音轨。
- 显示当前字幕文本。
- 展示处理版本号。
- 下载当前语言字幕和音轨。

#### 技术说明

浏览器原生多音轨能力支持不一致，MVP 建议使用自定义播放器：视频保持原画面轨，音频使用独立音频元素或 Web Audio 控制；字幕使用 WebVTT 渲染。生成的 MP4 多音轨文件作为下载产物，不作为网页预览唯一依赖。

### 3.6 结果下载页

#### 下载内容

- 原始视频。
- 原始音频。
- 分离后背景音。
- 分离后原人声。
- 每个目标语言的 SRT。
- 每个目标语言的 VTT。
- 每个目标语言的人声 TTS 分段音频。
- 每个目标语言的混合音轨。
- 可选 MP4 预览文件。
- manifest.json。

### 3.7 处理模板与 Skill 配置页

该页面面向内部管理员或高级运营，不作为普通运营的必经页面。

#### 必须支持

- 查看系统已启用的 Skill。
- 查看每个 Skill 当前使用的模型或供应商。
- 配置默认处理模板。
- 配置目标语言默认 TTS 音色。
- 配置 DeepSeek 翻译 prompt 版本。
- 查看最近 Agent Run 的失败原因。

#### MVP 不支持

- 不支持运营在页面上编写任意代码。
- 不支持 Agent 自主新增 Skill。
- 不支持绕过权限直接读取模型 API key。

## 4. 翻译产品要求

### 4.1 翻译风格

默认使用短剧本地化风格，不做直译。

要求：

- 台词口语化。
- 保留角色关系和情绪强度。
- 保留悬念、反转、冲突感。
- 避免中文式表达或英文式硬翻。
- 长句可拆短，适配配音节奏。
- 对专有名词、人物名、称呼保持一致。

### 4.2 翻译配置

| 配置 | 说明 |
| --- | --- |
| locale | 目标地区，例如 en-US、es-ES、es-MX、pt-BR。 |
| style | 短剧口语化、自然影视字幕、强情绪爽剧。 |
| glossary | 人名、地名、组织、固定译法。 |
| character_notes | 角色身份、语气、人设。 |
| forbidden_terms | 禁用词或平台敏感表达。 |
| length_policy | 尽量贴合原句时长，避免过长。 |

### 4.3 AI 翻译建议

MVP 可优先使用 DeepSeek 作为翻译模型。DeepSeek 只作为默认适配器，不能写死在业务逻辑中。接口层必须支持后续切换到其他 LLM。

## 5. 音频产品要求

### 5.1 BGM 保留

MVP 目标是保留原视频背景音、环境声和音乐，仅替换人声。实现上需要进行声源分离：

- 输出 background_track。
- 输出 source_vocal_track。
- TTS 生成 target_vocal_track。
- 将 target_vocal_track 与 background_track 混合为 target_mix_track。

### 5.2 质量边界

以下场景不保证高质量：

- 背景音乐过大，盖住人声。
- 多人同时说话。
- 原视频含强混响、强噪声。
- 人声和背景音高度混合。
- 角色低声耳语或喊叫失真。

### 5.3 TTS 要求

MVP 需要支持：

- 按目标语言生成配音。
- 按 speaker_id 选择音色。
- 设置语速。
- 尽量贴近原句时长。
- 单句失败可重试。
- 支持供应商切换。

MVP 不要求：

- 精准复刻原演员声音。
- 自动生成商业授权音色。
- 唇形级同步。
- 情绪完全匹配原表演。

## 6. 权限与审计

MVP 面向内部运营，可先使用简单账号体系或企业内部登录。

必须记录：

- 上传人。
- 上传时间。
- 原文件名和文件哈希。
- 调用模型类型。
- 生成版本。
- 人工修改记录。
- 下载记录。

## 7. 验收标准

### 7.1 主链路验收

- 上传 1 条 1-3 分钟中文或英文无字幕短剧。
- 选择 2 个目标语言。
- 系统能生成每个目标语言的字幕和混合音轨。
- 网页能切换字幕和音轨预览。
- 能下载结果包。
- 任一句译文修改后，可重跑对应语言字幕和 TTS。

### 7.2 质量验收

- 字幕时间轴基本跟随对白。
- 翻译语气符合短剧口语化表达。
- TTS 音轨与画面节奏基本一致。
- 原 BGM 在目标语言混音中可感知保留。
- 失败时能看到明确错误和重试入口。

### 7.3 非功能验收

- 任务失败不影响原视频和已完成产物。
- 重试不会生成不可追踪的脏数据。
- 所有生成产物都有版本号。
- 模型 API key 不暴露到前端。
- 上传文件和生成文件可按项目清理。
